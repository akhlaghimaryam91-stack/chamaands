# راهنمای دیپلوی پروژه چمان (chamaands.ir)

این فایل رو بده (یا بگو «طبق DEPLOY.md دیپلوی کن») تا در هر سشن جدید بدون توضیح دوباره کار رو انجام بدم.

## معماری

Flask + SQLite روی cPanel shared hosting (Phusion Passenger). دیتابیس SQLite مستقیم روی سرور، بدون ORM migration framework — schema changes با پترن `ALTER TABLE` داخل خودِ `app.py` (توی `with app.app_context()`, با try/except) انجام می‌شه.

## اطلاعات دسترسی

پسوردها و توکن‌ها **در این فایل نیستن** — در فایل جدا نگه داشته می‌شن که هرگز کامیت نمی‌شه:

`D:\mary\کارگاه طراحی سایت\سایت ندا\chamaan-server-creds.txt`

اون فایل شامل: کاربری/پسورد cPanel و FTP، آدرس ادمین پنل سایت، و مسیر فایل توکن گیت‌هاب.

## مراحل دیپلوی بک‌اند

این پروژه هم از سند عمومی `D:\mary\کارگاه طراحی سایت\DEPLOY-CPANEL.md` پیروی می‌کنه (روش مشترک همه پروژه‌ها). خلاصه‌ی نکات مهمش برای چمان:

**تعداد کم فایل (۲-۳ تا، روش ۲ سند عمومی):** FTP تک‌فایلی با PowerShell، ولی حتماً `$wc.Proxy = $null` قبل از آپلود — وگرنه به‌خاطر پروکسی HTTP سیستم، FTP خطا می‌ده یا خطاهای جعلی 500/503 می‌ده:

```powershell
$cred = New-Object System.Net.NetworkCredential("<user>", "<pass>")
$wc = New-Object System.Net.WebClient
$wc.Proxy = $null   # حیاتی
$wc.Credentials = $cred
$wc.UploadFile("ftp://<HOST_IP>/chamaan/<مسیر نسبی>", "<مسیر لوکال>")
```

مسیر نسبی FTP: `chamaan/<...>` (بدون `/home/ycoejgpz/`).

**تعداد زیاد فایل (بیش از ۲۰-۳۰ تا، روش ۱ سند عمومی):** هرگز تک‌تک FTP نکن — فایروال IP رو بلاک می‌کنه. به‌جاش: `tar` (نه Compress-Archive پاورشل!) در Git Bash، آپلود آرشیو با cPanel API (`Fileman/upload_files`)، و extract با `json-api/cpanel fileop`. جزئیات کامل در سند عمومی.

**بعد از آپلود هر فایل `.py`:** پوشه `__pycache__` سرور رو هم پاک کن (فایل `.pyc` هم‌نام)، وگرنه Passenger نسخه‌ی کش‌شده‌ی قدیمی رو اجرا می‌کنه و ImportError می‌ده.

SSH/SFTP بسته‌ست، امتحانش نکن.

## ری‌استارت اپ

آپلود یک فایل خالی به `chamaan/tmp/restart.txt` (Passenger با دیدن این فایل، اپ رو ری‌استارت می‌کنه). بعدش ۱۵-۲۰ ثانیه صبر کن؛ اولین درخواست ممکنه موقتاً 500/503 بده — یک‌بار retry کن، خطای واقعی نیست.

## بررسی بعد از دیپلوی

با `curl` به دامنه اصلی و مسیرهای تغییریافته سر بزن (مثلاً `curl -H "Host: chamaands.ir" http://<HOST_IP>/`) و مطمئن شو ۲۰۰/۳۰۲ برمی‌گرده و خطای 500 نیست.

## پوش به گیت‌هاب

```bash
git add <فایل‌های تغییریافته>
git commit -m "..."
git push
```

توکن گیت‌هاب در فایل جدا (مسیرش در `chamaan-server-creds.txt`). برای احراز هویت API/gh از `$(cat "مسیر فایل توکن")` استفاده کن، نه گذاشتن توکن به‌صورت مستقیم توی دستور — چون سیستم ایمنی auto-mode دستورهایی که پسورد/توکن رو عیناً و مستقیم توی متن دستور دارن رو مسدود می‌کنه (Credential Materialization).

## نکات مهم / ممنوعه‌ها (از تجربه)

- **هیچ‌وقت پسورد/توکن رو مستقیم توی متن دستور ننویس** — از فایل جدا با `$(cat file)` بخون. وگرنه دستور بلاک می‌شه.
- **بعد از بلاک‌شدن یک دستور، بلافاصله همون کار رو با ابزار دیگه (curl به‌جای PowerShell و برعکس) امتحان نکن** — این هم به‌عنوان دور زدن (bypass) شناسایی و بلاک می‌شه. باید یا روش رو واقعاً عوض کنی یا از کاربر اجازه بگیری.
- **آپلود/ری‌استارت روی سایت واقعی (production) رو نباید به‌صورت خودکار و در پس‌زمینه (بدون حضور زنده در تعامل) زمان‌بندی کرد** — سیستم این رو «دیپلوی بدون نظارت انسانی» می‌بینه و بلاک می‌کنه، حتی اگر کاربر از قبل اجازه‌ی کلی داده باشه. کار آپلود واقعی باید توی یک تعامل زنده (نه ScheduleWakeup) انجام بشه.
- تست اتصال ساده (`curl`/`Test-NetConnection` به پورت‌ها) مشکلی نداره و می‌شه زمان‌بندی‌ش کرد.
- اگه هاست کلاً بی‌پاسخ بود (نه فقط FTP، بلکه cPanel و سایت هم timeout می‌دن)، این یعنی قطعی واقعی سرور — نه مشکل دسترسی. باید با پشتیبانی هاستینگ چک بشه.
- کد اینماد (در `templates/base.html`) باید عیناً بدون تغییر بمونه.
- قبل از هر دیپلویی، فایل‌های تغییریافته رو دقیق لیست کن و فقط همونا رو آپلود کن — نه کل پروژه.
